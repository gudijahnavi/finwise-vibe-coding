# SHOPEZ – E-Commerce Application
### Full Stack Development Internship – Project Repository

## About SHOPEZ
SHOPEZ is a full-stack e-commerce web application that lets customers browse products,
manage a cart, place orders, and track deliveries, while giving admins a dashboard to
manage products, orders, and users. Built end-to-end following the internship's
Problem → Idea → Requirements → Design → Development → Testing → Documentation → Demo lifecycle.

## Repository Structure

1. **Brainstorming & Ideation** – Team info, raw idea list, problem statement, empathy map
2. **Requirement Analysis** – Customer journey map, DFD, functional/non-functional requirements, tech stack
3. **Project Design Phase** – Problem-solution fit canvas, project proposal, solution architecture
4. **Project Planning Phase** – Sprint plan, product backlog / user stories (Jira-style)
5. **Project Development Phase** – Environment setup, prompt log, AI tools log, feature checklist
6. **Performance Testing** – Test cases, performance/load results, bug log
7. **Documentation & Demo** – SRS, API documentation, user manual, final project report
8. **Project Demonstration** – Demo script, storyboard, Q&A prep

Each folder contains filled-in deliverables for SHOPEZ. Replace/extend with your own
screenshots, diagrams, and GitHub links as the project progresses.
