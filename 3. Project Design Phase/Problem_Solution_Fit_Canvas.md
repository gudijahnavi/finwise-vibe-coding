# Problem–Solution Fit Canvas — SHOPEZ

| Block | Content |
|---|---|
| **Customer Segment** | Online shoppers (25–40 yrs) who value speed, trust, and order visibility |
| **Problem** | 1) Multi-step, friction-heavy checkout 2) No unified order tracking 3) Weak search/filter 4) Low trust in payment security |
| **Existing Alternatives** | Large marketplaces (feature-heavy, impersonal); small independent stores (fast but untrustworthy/poor UX) |
| **Solution** | SHOPEZ: streamlined 4-step checkout, real-time order status page, robust search/filter, visible security via reputable payment gateway integration |
| **Root Cause Check** | Is checkout friction the *symptom* of poor UX design, or a *root cause* of cart abandonment? → Root cause: unclear steps + no saved info + no trust cues. SHOPEZ addresses the root cause directly, not just a cosmetic fix. |
| **Key Metrics** | Cart-to-order conversion rate, checkout completion time, repeat purchase rate, order-status page visits (proxy for reduced support queries) |
| **Unique Value Proposition** | "Shop, pay, and track — all in one place, in under a minute." |
| **High-Level Feature List** | Auth · Catalog & Search · Cart · Checkout & Payment · Order Tracking · Admin Dashboard · Wishlist · Reviews |

## Fit Validation
The solution maps 1:1 onto each identified pain point from the Empathy Map and
Customer Journey Map (Phase 1 & 2) — confirming SHOPEZ addresses genuine friction
points rather than inventing unneeded features.
